CONTROLLI AUTOMATICI: Esercitazioni MATLAB

Prof. Giuseppe Oriolo 

Da questa pagina è possibile scaricare alcuni file utili per le esercitazioni di vari corsi nell'ambito dei Controlli Automatici. Generalmente si tratta di programmi MATLAB (*.m) o SIMULINK (*.mdl). Per ogni argomento, viene indicato il corso cui le esercitazioni si riferiscono (RM1 = Università di Roma "La Sapienza"; RM3 = Università di Roma Tre).

Informazioni su MATLAB e SIMULINK
Avvertenze sui file

I file sono in formato ASCII, e quindi sono utilizzabili su qualsiasi piattaforma di calcolo su cui siano installati MATLAB (in particolare Windows, MAC OS, Unix, Linux) e il Control System Toolbox. I file sono commentati e dovrebbero funzionare sulle versioni 4.x e successive di MATLAB. Alcuni file si riferiscono agli esercizi contenuti in:

L. Lanari, G. Oriolo: "Controlli Automatici - Esercizi di Sintesi", EUROMA-La Goliardica, 1997.

Analisi di sistemi dinamici lineari (RM1: C.A. I modulo)

  • Studio dell'evoluzione libera nello stato di sistemi bidimensionali con autovalori reali: la matrice dinamica viene generata interattivamente specificandone autovalori e autovettori (EvLib2D.m)
  • Studio dell'evoluzione libera nello stato di sistemi bidimensionali:
    • autovalori reali negativi (BiDimRealNegEig.m)
    • autovalori reali misti (BiDimRealMixEig.m)
    • autovalori complessi a parte reale negativa (BiDimCmplxNegEig.m)
    • autovalori complessi a parte reale positiva (BiDimCmplxPosEig.m)
  • Risposta libera e forzata del sistema massa-molla-smorzatore (MMS.m)
  • Risposta a regime permanente: uno schema Simulink per simulare la risposta forzata di sistemi asintoticamente stabili (con poli reali o complessi), semplicemente stabili o instabili a ingressi a gradino unitario, a rampa unitaria, sinusoidali (RegPerm.mdl)

Assegnazione degli autovalori con lo spazio di stato (RM1: C.A. II modulo; RM3: C.A. I)

Regolazione asintotica dell'uscita (RM3: C.A. II)

  • Un esempio di regolazione dell'uscita con informazione completa: doppio integratore in assenza di disturbi (cfr. slides "Regolazione asintotica dell'uscita", Esempio 1). Due implementazioni SIMULINK:
DoubleInt1.mdl: uno schema causale (cioè che non usa derivate del segnale di riferimento); derivato dalle equazioni del regolatore, funziona solo con il riferimento per cui e' progettato
 
DoubleInt2.mdl: uno schema non causale (cioè che usa derivate del segnale di riferimento); derivato dalla stabilizzazione del sistema d'errore, funziona con qualsiasi riferimento che sia derivabile due volte. Tuttavia, a causa del ritardo introdotto dal derivatore (che MATLAB implementa "all'indietro") non riesce a inseguire riferimenti che variano rapidamente (come esponenziali veloci o sinusoidi con frequenza elevata).
  • Movimentazione di un carrello meccanico in presenza di attriti e disturbi: regolazione asintotica con informazione completa e con reazione dall'errore (cfr. slides "Regolazione asintotica dell'uscita", Esempi 2 e 3). Un file MATLAB e due schemi SIMULINK:
CarrelloInit.m: file di inizializzazione (contiene i valori dei parametri e varie elaborazioni per determinare i controllori)
 
CarrelloIC.mdl: uno schema con informazione completa
 
CarrelloRE.mdl: uno schema con reazione dall'errore
  • Controllo dell'estremità (tip) di un braccio robotico flessibile: regolazione asintotica in assenza di disturbi per un modello lineare con 3 modi di deflessione (cfr. slides "Controllo di un robot flessibile"). Un file MATLAB e uno schema SIMULINK:
Flex3RegTipInit.m: file di inizializzazione (contiene i valori dei parametri e varie elaborazioni per determinare i controllori)
 
Flex3RegTip.mdl: uno schema di regolazione con informazione completa

Teoria della stabilità non lineare (RM3: C.A. II)

  • Un esempio di sistema non lineare con un punto di equilibrio instabile, ma con traiettorie non divergenti: l'oscillatore di Van der Pol (cfr. slides "Teoria della stabilità").
Schema SIMULINK: VanderPol.mdl
  • Un esempio dovuto a Vinograd di un sistema non lineare con un punto di equilibrio instabile, ma cui convergono asintoticamente tutte le traiettorie (cfr. slides "Teoria della stabilità").
Schema SIMULINK: Vinograd.mdl

Stabilizzazione via retroazione dallo stato (RM3: C.A. II)

  • Un esempio di stabilizzazione di sistema non lineare mediante approssimazione lineare: il pendolo controllato mediante una coppia al giunto (cfr. slides "Stabilizzazione via retroazione dallo stato").
PendulumTorqueInit.m: file di inizializzazione (contiene i valori dei parametri meccanici e del controllore)
 
PendulumTorque.mdl: schema SIMULINK di simulazione
  • Un esempio di stabilizzazione di sistema robotico non lineare: un robot 2R in moto nel piano verticale (cfr. slides "Controllo dei robot manipolatori"). Due schemi: controllo PD + compensazione gravità (per il set-point) e controllo via linearizzazione esatta (valido sia per il set-point che per il tracking).
Robot2RInit.m: file di inizializzazione (contiene i valori dei parametri meccanici e dei due controllori)
 
Robot2RPD: schema SIMULINK di simulazione con controllo PD + compensazione gravità (set-point)
 
Robot2RLE: schema SIMULINK di simulazione con controllo via linearizzazione esatta (set-point o tracking)
