Istruzioni per l'uso di FLEX e BISON in ambiente DJGPP

 Nota bene: queste istruzioni valgono solo per l'uso di FLEX e BISON con DJGPP. Sotto altri sistemi operativi (es. UNIX, LINUX) cambiano i nomi dei file generati da FLEX e BISON (es. lexyy.c diventa lex.yy.c)

  1. Creare il file file1 di specifica FLEX dell'analizzatore lessicale.
  2. Creare il file file2 di specifica BISON dell'analizzatore sintattico. Si consiglia di usare l'estensione standard ".y" per i file contententi specifiche BISON.
  3. Generare l'analizzatore lessicale in C mediante il comando:

    4. flex file1

    (viene generato il file lexyy.c);

  4. Generare l'analizzatore sintattico in C mediante il comando:

    5. bison -d file2

    (vengono generati i file file2_tab.c e file2_tab.h);

    NOTA: L'opzione -d di BISON serve a generare l'header file. Tale file deve essere incluso dal sorgente lexyy.c generato da flex. Perciò questa dichiarazione di inclusione va messa nella parte dichiarazioni del file file1 di specifica FLEX.

  5. Compilare questi moduli C mediante il comando:

    6. gcc lexyy.c file2_tab.c -lfl -o pascom.exe

    (viene generato l'eseguibile pascom.exe)

    NOTA: L'opzione -lfl serve a collegare i moduli generati dal compilatore con la libreria di FLEX.

  6. Richiamare l'analizzatore sintattico mediante il comando:

    7. pascom < filesorgente

    (viene generato in standard output l'albero sintattico risultante dall'analisi).

    NOTA: L'albero sintattico può venire scritto su file ridirigendo l'output nel seguente modo:

    pascom < filesorgente > filedestinazione
