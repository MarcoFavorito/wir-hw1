Merging vs. Splitting

  • Merging

    • Large increase in memory when sub-entities have many attributes
    • Useless if the main entity has few attributes and/or relationships (useful in the opposite case)
    • Null values

  • Splitting

    • No wasting of memory if there is no overlap between the sub-entities
    • Useless if there are many joint accesses to the sub-entities
    • Need of integrity checking procedures
