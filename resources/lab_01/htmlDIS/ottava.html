Ottava esercitazione autoguidata

    Simulazione di (parte di) una prova di esonero


Per questa esercitazione bisogna disporre di un dischetto personale. Si tratta di un allenamento per l'esonero e pertanto ci si aspetta che ciascuno tenti di risolvere l'esercizio da solo.

La prova di esonero comporterà la soluzione di tre problemi. In questa esercitazione possiamo allenarci usando due problemi, divisi nelle seguenti sezioni.

Attivare il TurboPascal. Scegliere il comando Dos Shell dal menù File.

Posizionarsi sulla directory di rete E:\PROGRAM\FOND\TEMPERIN\AUTOGUID\OTTAVA\.

opure E:\program\fond\daloisi\AUTOGUID\OTTAVA per gli elettronici
 

Copiare sul dischetto, nella directory "radice" A:\, i file contenuti in questa directory.

Posizionarsi sul driver del dischetto (A:) e tornare al TurboPascal, inviando il comando EXIT.
 
 

Primo problema

Ho detto di posizionarsi su A: prima di rientrare in turbopascal.
 
 

ESO1: File di interesse per questo problema

Il file ESO1.PAS contiene un programma quasi completo che manipola una matrice e un vettore.

Il file SOL1.PAS è attualmente vuoto e dovrà contenere la soluzione del problema (una procedura ESERCIZIO1)

I file INPUT1.1,input1.2, INPUT1.3, INPUT1.4 contengono dati di prova per testare il programma (e la procedura).

Il file SOLUZ8/ESOL1.PAS contiene unapossibile soluzione al problema (cioè una possibile procedura ESERCIZIO1).
Si presume che non venga visto fino alla fine ....

Il programma in ESO1.PAS contiene le dichiarazioni per le costanti N ed M e per i tipi
 

Type MATRICE = array[1..N,1..M] of integer

Type VETTORE = array[1..N] of integer;


Richieste dell'esercizio
 

Bisogna scrivere nel file SOL1.PAS la procedura ESERCIZIO1 che, ricevendo per parametri una matrice di tipi MATRICE e un array di tipo VETTORE riempia il vettore in modo che il suo elemento i-esimo contenga la somma degli elementi della riga i-esima della matrice.
 

Esecuzione dell'esercizio

È bene dare un'occhiata al contenuto del file ESO2.PAS. Per la soluzione dell'esercizio bisognerà dichiarare opportunamente la funzioneESERCIZIO2 nel file SOL2.PAS. Questo file viene automaticamente incluso nel programma, che così può usare la procedura.

Attenzione a non modificare il programma in ESO2.PAS. Il file ESO2.PAS deve rimanere esattamente com'è.

Le uniche modifiche ammesse sono quelle al file SOL2.PAS. Non mettete le mani altrove!

Una volta scritta la procedura nel file SOL1.PAS, trasferirsi su ESO1.PAS, compilare e fare le esecuzioni di prova.

Ad ogni esecuzione il programma chiede l'immissione di un nome di file (uno dei file di dati di prova input1.k, oppure un file costruito personalmente Ö), riempie la matrice mat con i dati in esso contenuti, chiama la procedura ESERCIZIO1 e poi stampa il contenuto del vettore vet.

Idealmente, dopo aver testato il programma e visto che la procedura prodotta funziona bene, si passa al successivo esercizio. Il lavoro conservato in SOL1.PAS è pronto per la correzione.
 

       
ESO2: Secondo problema

File di interesse per questo problema

Il file ESO2.PAS contiene un programma quasi completo che manipola una matrice.

Il file SOL2.PAS è attualmente vuoto e dovrà contenere la soluzione del problema (una funzione ESERCIZIO2)

I file INPUT2.1,input2.2, INPUT2.3, INPUT2.4 contengono dati di prova per testare il programma (e la procedura).

Il file SOLUZ8/ESOL2.PAS contiene unapossibile soluzione al problema (cioè una possibile procedura ESERCIZIO2).
Si presume che non venga visto fino alla fine ....

Il programma in ESO2.PAS contiene le dichiarazioni per le costanti N ed M e per i tipi

Type TipoMATRICE = array[1..N,1..M] of integer

         
         
Richieste dell'esercizio

Bisogna scrivere nel file SOL2.PAS la funzione ESERCIZIO2 che, ricevendo una variabile file contenente almeno NxM numeri interi (in sequenza) e 4 indici h1, h2, k1, k2, riempie un array mat di tipo TIPOMATRICE con gli interi del file e restituisce il massimo valore contenuto nella sottomatrice avente come vertice superiore sinistro l'elemento mat[h1,k1] e come vertice inferiore destro l'elemento mat[h2,k2].

Attenzione, il programma principale consegna alla funzione un file già "resettato" per la lettura, quindi la funzione non deve eseguire reset.
 

Esecuzione dell'esercizio

È bene dare un'occhiata al contenuto del file ESO2.PAS. Per la soluzione dell'esercizio bisognerà dichiarare opportunamente la proceduraESERCIZIO2 nel file SOL2.PAS. Questo file viene automaticamente incluso nel programma, che così può usare la procedura.

Attenzione a non modificare il programma in ESO2.PAS. Il file ESO2.PAS deve rimanere esattamente com'è. Le uniche modifiche ammesse sono quelle al file SOL2.PAS.

Una volta scritta la procedura nel file SOL2.PAS, trasferirsi su ESO2.PAS, compilare e fare le esecuzioni di prova.

Ad ogni esecuzione il programma

  • chiede l'immissione di un nome di file (uno dei file di dati di prova input2.k, oppure un file costruito personalmente Ö ma prima di costruire nuovi file di input verificare come sono fatti quelli forniti: )
  • legge dal file i quattro numeri che saranno gli indici h1, h2, k1, k2
  • chiama la funzione ESERCIZIO2 passandole quel che deve passarle
Idealmente, dopo aver testato il programma e visto che la procedura prodotta funziona bene, si passa al successivo esercizio. Il lavoro conservato in SOL2.PAS è pronto per la correzione.
