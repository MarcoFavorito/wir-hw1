  2. Classi base astratte e funzioni virtuali pure

    3. La progettazione può portare alla definizione di classi che presentano un livello di astrazione talmente elevato che non ha senso prevedere istanze concrete. Queste classi vengono usate soltanto come base concettuale per la derivazione di gerarchie di classi (ad esempio la classe Object o Applet dei package java forniti da Sun). Per classi di questo tipo può risultare inutile l'implementazione di alcuni metodi in quanto destinati ad essere ridefiniti nelle classi derivate. Il C++ permette di definire un metodo "virtuale" senza specificarne il corpo

    virtual void G() = O;

    Una classe che contiene almeno una funzione virtuale pura, per la quale non è quindi possibile definire oggetti, viene detta classe base astratta.

  3. Relazione is-a tra due classi

Se la classe B deriva da A, risulta naturale poter utilizzare oggetti di tipo B in ogni contesto in cui è lecito utilizzare oggetti di tipo A.
