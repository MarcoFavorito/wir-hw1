This lecture consists of two parts. System architecture of RoboCup Rescue Simulator and fundamental components - city map, simulators and agents - are introduced in the first part. This part aims that you can run simulations, make your own maps, to program simulators and agents. And also tools will also be presented. The tools have developed or been developed by teams that participated in past competitions. Applying this Rescue Simulators to practical uses, we have many comments from various fields. They are needs from industry and research seeds, for example, size of simulation, model of disasters and human behaviors, interactions between real world, so on. Linkage with global (city size) simulation and local (house size) simulation is one of possible solution. Let's discuss them as topics in a new league.

Plan for practicals:

  • 1st practice: you can use RoboCup Rescue simulations.
    1. install rescue simulation environment.
    2. run simulations;
      • at Kobe city and other cities.
      • with different conditions.
      • with different disaster simulators.
    3. check simulation results using off-line log viewer.

  • 2nd practice: you can use them as a platform of your research and make your own agents.
    1. create maps.
      • map format.
      • map editor.
      • map generator.
    2. use tools for development.
      • API for programming agents.
      • autorun system.
      • 3D viewer.
    3. discussions
