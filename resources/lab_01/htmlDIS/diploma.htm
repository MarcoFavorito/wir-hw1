L'esame consiste nella realizzazione di un progetto da scegliersi tra i seguenti:

Progetto numero 1.  Implementazione di un server  FTP.

L'obiettivo di tale progetto e' l'implementazione di un  client FTP e di un server  FTP multithreaded in grado di implementare le seguenti funzionalita':

  • get: comando che permette di trasferire un file remoto nella directory locale al Client (comando RETR del protocollo);
  • put: comando che permette di trasferire un file locale nella directory locale al server (comando STOR del protocollo);
  • dir: comando che permette di ottenere la lista dei files nella directory remota (comando LIST del protocollo);
  • quit:  per chiudere la connessione (comando QUIT del protocollo).
     

Si vogliono inoltre implementare i seguenti comandi del protocollo:

  • USER: per acquisire il nome dell'utente;
  • PORT: per comunicare la porta su cui trasferire i dati;
  • CWD:  per cambiare la directory;
  • PWD:  per conoscere la directory di lavoro;
  • DELE: per cancellare un file remoto.

Nel definire l'interazione tra il Client FTP ed il  Server scritto dall'utente, si vogliono osservare le raccomandazioni rfc 959che definiscono il protocollo FTP.
 

Progetto numero 2.  Realizzazione di un Sistema  SMTP.


 Si vuole sviluppare un  sistema  Cleint/Server SMTP (Simple Mail Transfer Protocol) per l'invio della posta elettronica secondo lo standard  RFC0821. Il server SMTP dovra' essere in comunicazione con un utente locale attraverso un'applicazione per la posta elettronica.

 Il server SMTP dovra' svolgere le seguente funzionalita':

  1. Ricevere ed inviare i messaggi da un' applicazione  per la posta elettronica di un utente locale.
  2. Salvare i messaggi provenienti da altri utenti diretti ad un utente locale in un file secondo il formato richiesto da un'applicazione per la posta elettronica (ex : POP3 RFC1939).

Il Client SMTP dovra' svolgere le seguenti funzionalita':

  1. Risolvere gli indirizzi degli utenti di destinazione attraverso l'accesso al DNS locale.
  2. Recapitare i messaggi al SMTP server dell'utente di destinazione.

 

Progetto numero 3.  Gestione di un Sistema  di Talk multiutente.

L'obiettivo di tale progetto e' lo sviluppo di un'applicazione Clent/Server che permetta la realizzazione di un talk multiutente
Si vogliono implementare le seguenti funzionalita':
 

  1. Registrazione/Deregistrazione del Client presso il Server centrale.
  2. Richiesta al Server da parte del Client degli utenti disponibili per lo stabilimento di un talk e della lista dei talk in corso.
  3. Richiesta attraverso il server centrale di stabilire un nuovo talk con un Client o di entrare in un talk gia' in corso.
  4. Richiesta dei talk in corso presso altri client e lista dei client partecipanti in ogni talk.
  5. Replica del Client attraverso il Server centrale alle richieste di stabilimento di un talk.  La replica sara' di 4 tipi diversi.
  • Rifiuto allo stabilimento del Talk.
  • Inclusione dell'utente  nel talk indicato dal client;
  • Avvio di un nuovo Talk;
  • Disposizione dell'utente in coda di attesa fino al termine di uno dei talk in corso per una richiesta di stabilimento di un nuovo talk.
  1. Ognuno dei  talk attivi presso un Client dovra' svolgersi in una finestra separata.  All'interno di ogni finestra di Talk, ad  ognuno dei Client e' dedicato uno spazio apposito per la scrittura.
     


 

Progetto numero 4.  Proxy HTTP con gestion di cache e file di log.

L'obiettivo di tale progetto e' lo sviluppo di un Proxy HTTP. Tale Proxy intercetta le chiamate del browser ad una pagina P. Verifica la disponibilità nella propria cache della pagina P. Se P è in cache la pagina viene ritornata dal Proxy, altrimenti il Proxy si connette al server Web contenente P, scarica la pagina in locale, inserisce P in cache e infine ritorna P al browser richiedente. La cache viene gestita secondo la politica LRU. Ogni oggetto richiesto viene archiviato in un file di log.
