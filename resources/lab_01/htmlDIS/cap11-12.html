Prima esercitazione di laboratorio

È opportuno portare all'esercitazione almeno un dischetto, sul quale salvare i programmi 90 scritti.




Obiettivi di questa esercitazione:

  1. Prendere familiarità con il sistema operativo Windows NT.
  2. Visitare la pagina Web del corso.
  3. Prendere familiarità con il compilatore ELF 90.
  4. Prendere familiarità con l'editor.


Sistema operativo Windows NT

  1. Operazioni preliminari di accensione della macchina e accesso al sistema.
    1. Accendere, se necessario, la macchina, ed attendere (senza premere alcun tasto) che compaia una finestra di accesso.
    2. Quando richiesto, premere contemporaneamente i tasti <CTRL ALT CANC>.
    3. Nella finestra che compare, digitare nella prima riga

      user

      lasciare la seconda riga vuota, e premere <INVIO>.

  2. Creazione di un proprio direttorio di lavoro.
    1. Lanciare il programma ``Gestione risorse'', facendo click su AVVIO (in basso a sinistra) e poi scegliendo ``Programmi'' e ``Gestione risorse''.
    2. Creare un proprio direttorio, eseguendo i passi:

      1: ``File'', 2: ``Nuovo'', 3: ``Cartella'',

      come sottodirettorio di C:\Temp, dandogli, ad esempio, il nome Miodir.

  3. Alcune nozioni utili per lavorare meglio in Windows NT.
    • È possibile rendere il programma ``Gestione risorse'' temporaneamente inattivo, facendo click sul quadratino più a sinistra fra quelli in alto a destra.
    • È apparso nella parte inferiore dello schermo un rettangolo con il nome ``Gestione risorse''. Facendo click su esso, è possibile ``portare in primo piano'' il programma, rendendolo attivo.
    • Windows NT consente di usare contemporaneamente più programmi (ad es. ``Gestione risorse'' e ``Netscape'').
    • È possibile alternare fra i vari programmi usati durante una sessione di lavoro con le due operazioni di cui sopra.

Pagina Web del corso

  1. Attivazione del browser e accesso alla pagina Web del corso.
    1. Attivare il browser Netscape, facendo doppio click sulla sua icona.
    2. Digitare l'indirizzo della pagina del corso nell'apposito spazio accanto a ``location''; premere <INVIO>.

      Indirizzi delle pagine Web di interesse per il nostro corso:

      • Home page del corso di Informatica (Ing. Aerospaziale):

        http://www.dis.uniroma1.it/~cadoli/didattica/informatica

      • Home page del laboratorio di Via Tiburtina

        http://tibur.dis.uniroma1.it

      NOTA: il carattere `~' (tilde) si ottiene digitando <ALT 126> (126 va digitato sul tastierino numerico sulla destra, e la spia BLOC NUM deve essere accesa).

  2. Ispezione del materiale del corso.
    1. Facendo click con il mouse sul testo sottolineato, si possono seguire i collegamenti di interesse. Per tornare alla pagina precedente, fare click sull'icona ``Back''.
    2. Trovare il direttorio dell'esercitazione odierna.
    3. Copiare il programma 90 CIAO.F90 nel proprio direttorio di lavoro (C:\Temp\Miodir), utilizzando il menu ``File'' e l'opzione ``Save As''.


Compilatore ELF 90

  1. Operazioni preliminari di cambio direttorio.

    1. Lanciare il programma ``Prompt dei comandi'', facendo click su AVVIO (in basso a sinistra) e poi scegliendo ``Prompt dei comandi''.

    2. Cambiare direttorio di lavoro, digitando:

      cd Temp\Miodir

  2. Compilazione, collegamento ed esecuzione.
    1. Compilare il file sorgente CIAO.F90, digitando:

      elf90 ciao.f90

      Ciò provoca la creazione del file oggetto CIAO.OBJ.

    2. Collegare il file oggetto CIAO.OBJ, digitando:

      elf90 ciao.obj

      Ciò provoca la creazione del file eseguibile CIAO.EXE.

    3. Mandare in esecuzione il file eseguibile CIAO.EXE, digitando:

      ciao

Editor

  1. Operazioni preliminari di cambio direttorio e attivazione del programma.
    1. Lanciare il programma ``Prompt dei comandi'', facendo click su AVVIO (in basso a sinistra) e poi scegliendo ``Programmi'' e ``Prompt dei comandi''.

      NON eliminare la finestra ``Prompt dei comandi'' usata per il compilatore.

    2. Cambiare direttorio di lavoro, digitando:

      cd Temp\Miodir

    3. Lanciare il programma di editing, digitando

      edit

    4. Premendo contemporaneamente i tasti ALT e Enter si ottiene una finestra a tutto schermo. Premendo Esc si ritorna alla dimensione precedente.

  2. Scrittura di un programma 90 e salvataggio del file sorgente.
    1. Scrivere il programma 90 assegnato per l'esercitazione.

      È possibile muoversi nel testo usando il mouse e/o i tasti freccia.

    2. Salvare il programma 90, premendo <ALT F> scegliendo `S'alva e digitando il nome scelto per il file sorgente, che deve avere estensione `.F90'.

      In alternativa, può essere usato il mouse.

  3. Compilazione del file sorgente scritto.
    1. Vedere il paragrafo 11.3 (sostituendo il nome ciao.f90) con il nome del file prescelto).

    2. Nel caso in cui siano presenti errori, è opportuno che il comando di compilazione sia

      elf90 <nomefile>.f90 | more

      In questa maniera è possibile vedere il numero della linea dove occorre ciascun errore.

      Premendo la barra spaziatrice o <INVIO> è possibile vedere tutti gli errori (molto spesso, tuttavia, solo il primo errore evidenziato è significativo).

      NOTA: il programma di editing visualizza, nell'angolo in basso a destra, la riga e la colonna del cursore.

    3. È opportuno ricompilare il file sorgente dopo avere corretto il primo errore e salvato il file sorgente.

    4. Altri aspetti del compilatore sono presentati nel capitolo 12.
  4. Uscita dall'editor.

    Dopo avere terminato la scrittura di tutti i programmi 90, si può uscire dall'editor premendo <ALT F> e scegliendo `E'sci.

    In alternativa, può essere usato il mouse.

Salvataggio dei propri programmi 90 su dischetto

  1. È importante salvare i propri programmi 90 su un dischetto, perché i dischi rigidi del laboratorio vengono periodicamente cancellati.

  2. Mediante il programma ``Gestione risorse'':
    1. copiare tutti i programmi 90 dal proprio direttorio di lavoro,
    2. incollarli sul proprio dischetto.

  3. CONSIGLIO: strutturare il proprio dischetto in sottodirettori (cfr. paragrafo 11.1), uno per argomento e/o per esercitazione.

Note sul compilatore usato

Tutti i programmi presentati in questa dispensa sono stati compilati e verificati con il compilatore Essential Lahey Fortran 90 Compiler Release 4.00b.

In questo capitolo vengono illustrate alcune differenze fra il compilatore e l'esposizione del 90 presentata nel libro di testo. Vengono inoltre chiariti alcuni aspetti pratici sulla compilazione di programmi strutturati in più unità di programma. Infine, vengono riportate le istruzioni per l'installazione del compilatore sul proprio calcolatore personale.

Differenze rispetto al libro di testo

Rispetto a quanto specificato nel libro di testo, le principali differenze riscontrate nel compilatore usato sono le seguenti:
  • L'istruzione IMPLICIT NONE è obbligatoria in tutte le unità di programma.

  • L'istruzione STOP è obbligatoria al termine di un programma.

  • Il nome del programma è obbligatorio nella linea END PROGRAM.

    Lo stesso vale per END FUNCTION, END SUBROUTINE ed END MODULE.

  • Il costrutto IF non può avere nomi (cfr. libro di testo, pag. 119).

  • Se una riga termina con `&', la riga successiva deve obbligatoriamente iniziare con `&' (cfr. libro di testo, pag. 14).

  • Una dichiarazione del tipo 
    CHARACTER :: iniziale
    (cfr. libro di testo, pag. 23) non viene compilata, e va sostituita con la seguente 
    CHARACTER(1) :: iniziale

  • Subroutine e funzioni devono essere necessariamente definite all'interno di MODULE, come indicato nel testo, al paragrafo 6.5.

  • Una dichiarazione del tipo 
    INTEGER FUNCTION romano2intero (ch)
    (cfr. libro di testo, pag. 366) non viene compilata, e va sostituita con le seguenti 
    FUNCTION romano2intero (ch)
INTEGER :: romano2intero

  • Una funzione non deve essere ridichiarata nell'unità di programma che la usa.

  • Un argomento di tipo array deve essere necessariamente di forma presunta (cfr. paragrafo 6.5 del libro di testo). Non sono quindi ammesse dichiarazioni come quelle di pagg. 326, 327.

  • La specifica INTENT INOUT non è ammessa; occorre usare INTENT IN OUT (cfr. pag. 325 del libro di testo).

  • L'attributo INTENT è obbligatorio per tutti gli argomenti passati ad un'unità di programma.

  • Gli argomenti fittizi di una FUNCTION devono necessariamente avere l'attributo INTENT IN.

  • Il descrittore di formato `X' non è ammesso.

Compilazione di programmi strutturati in più unità di programma

Il compilatore permette di compilare programmi strutturati in più unità di programma, in unico file o diversi file, come evidenziato nei capitoli  .

A tal fine, è necessario lanciare il comando di compilazione per ognuno dei file in cui è strutturato il programma. Ad esempio, supponendo che il programma sia strutturato nel seguente modo (cfr. paragrafi  e ):

  • un file ROM2INT.F90 che contiene un'unità MODULE, e

  • un file CLIROM2I.F90 che contiene l'unità principale PROGRAM,
occorre lanciare i comandi

elf90 rom2int.f90

e

elf90 clirom2i.f90


Quando sono stati corretti tutti gli errori nei file sorgente, si può procedere al collegamento. L'unico comando necessario è

elf90 clirom2i.obj


L'esecuzione del programma si ottiene lanciando il comando

clirom2i

Procedura d'installazione di Elf90 su Windows95/98/NT/2000

Nel caso in cui si desideri installare il compilatore freeware Essential Lahey Fortran 90 Compiler Release 4.00b sul proprio calcolatore, si raccomanda di seguire le seguenti istruzioni:

  1. Scaricare il file f.exe dalla pagina web del corso.

  2. Eseguire il programma di installazione cliccando sull'icona del file, oppure digitando elf90 seguito da Invio dopo aver aperto una finestra DOS dalla posizione in cui è stato memorizzato il file sul disco rigido.
  3. La collocazione sul disco rigido suggerita dal programma di installazione è c:\elf90. Per accettare tale scelta cliccare su Unzip. Affinché l'installazione si possa considerare conclusa, deve apparire la scritta 30 files unzipped successfully. Cliccare su OK e Close per uscire dalla finestra di autoestrazione di WinZip.
  4. È ora necessario includere il direttorio c:\elf90 nel proprio path.
    1. Windows95/98
      • Aprire il file autoexec.bat utilizzando un qualsiasi editor di testo (EDIT, NotePad, WordPad, etc. )
      • Aggiungere il direttorio c:\elf90 alla fine della propria linea di dichiarazione del path: 
             SET PATH ...........;c:\elf90
        Ogni direttorio dichiarato nel path dovrebbe essere separato da ``;'' (senza spazi!)
    2. WindowsNT/2000
      • Cliccare con il tasto destro del mouse sull'icona Risorse del Computer , selezionare Proprietà e poi Avanzate .
      • Cliccare su Variabili d'ambiente... , tra le Variabili di Sistema selezionare Path e quindi Modifica . Alla fine del path esistente, aggiungere 
         ;c:\elf90
        e salvare la modifica apportata selezionando OK nelle finestre ancora aperte.
  5. Riavviare il computer per eseguire il nuovo path.
  6. Testare l'installazione su un programma test (test.f90) digitando da un prompt di DOS:
    elf90 test.f90
    Se la compilazione è avvenuta senza la comunicazione di errori eseguire il programma creato digitando:
    test
  7. Si noti che il compilatore crea, in alcune circostanze, file con varie estensioni (ad es., `.MAP', `.MOD', ...), che non devono essere cancellati.
