Editor Visuali e Gestori di Siti (EGS)

WYSIWYG

Funzionalità per la creazione delle pagine

Strumenti di supporto alla creazione automatica di link

Strumenti di supporto alla gestione (controllo Link) 

Es. Front Page, SiteMill

Poco scalabile e poco adatto ad applicazioni di dimensioni  medio-grandi

Sommario Caratteristiche (EGS)

  • Ciclo di vita: Implementazione, Mantenimento
  • Output: HTML statico, comandi navigazione
  • Astrazioni: pagine, stili di presentazione
  • Architettura: 2 strati
  • Contenuto: nessuno
  • Navigazione: Buono, strutture di navigazione
  • Presentazione: Buono, uso di stili

Strumenti per lo sviluppo di applicazioni ipermediali (IPM)

Uso di metafore di organizzazione (es. libro)

Generazione automatica di strutture di navigazione

Connettività con Data Base

Connettività con il Web

Es. Toolbook, Authorware

Nessun supporto per il ciclo di vita, architettura inadeguata

Sommario Caratteristiche (IPM)

  • Ciclo di vita: Progetto (authoring in the large), Implementazione
  • Output: HTML /JAVA, oggetti multimediali, comandi navigazione
  • Astrazioni: metafore di documenti
  • Architettura: 2 strati 3 strati (ma non connessi)
  • Contenuto: nessuno
  • Navigazione: Molto Buono, strutture di navigazione, ricerca
  • Presentazione: Ottimo, grafica accurata

Integratori Web-DBPL (WDB)

Separazione del contenuto dalla presentazione e dalla navigazione

Estensioni di HTML (Es. ASP) 

Estensioni di DB (Es. Web Toolkit Oracle)

Estensioni di JAVA

Scarso supporto metodologico, in particolare alla navigazione e presentazione, orientati alla programmazione

Sommario Caratteristiche (WDB)

  • Ciclo di vita: Progetto (contenuto), Implementazione, Manutenzione (contenuto)
  • Output: HTML come risultato di interrogazioni al DB (dinamico), supporto per l'accesso al DB 
  • Astrazioni: strutture di DB, pagine, interfacce ad oggetti
  • Architettura: 3 strati (creazione dinamica delle pagine)
  • Contenuto: molto buono basato su tecniche DB
  • Navigazione: scarso
  • Presentazione: scarso

Web form editors e supporto DB per la pubblicazione (DBS)

La programmazione esplicita è sostituita da varie forme di generazione automatica di codice

Esportazione dati dal DB

Gestione di documenti (report)

Sviluppo di applicazioni basato su form

Es.Developer 2000 (Oracle), Microsoft Acces, PowerBuilder

Adeguati per applicazioni INTRANET (porting su interfaccia Web di applicazioni client-server), ma limitati dal punto di vista grafico e della presentazione.

Sommario Caratteristiche (DBS)

  • Ciclo di vita: Implementazione, Testing, Manutenzione 
  • Output: HTML (+ Java) o formati proprietari ActiveX, VBscript, Javascript, (interpretati con plug-in) 
  • Astrazioni: forms, reports, strutture DB
  • Architettura: molti strati (collegamento ad applicazioni distribuite)
  • Contenuto: molto buono basato su tecniche DB
  • Navigazione: limitato, legato alla struttura del DB
  • Presentazione: rigido, basato su schemi tipo

Strumenti multiparadigmi (MUP)

EGS + WDB

IPM + WDB

EGS + DBS

IPM + DBS

Interoperabilità può essere un problema

Generatori di applicazioni (GEN)

Strumenti che coprono l'intero processo di sviluppo.

Modello concettuale basato su documenti (Hyperwave IM)

Modello concettuale basato sui dati (Developer 2000)

Il primo è migliore per il supporto alla navigazione, il secondo per il supporto nella gestione dei dati

Sommario Caratteristiche (GEN)

  • Ciclo di vita: Concettualizzazione, Progetto, Implementazione, Testing, Manutenzione 
  • Output: HTML 
  • Astrazioni: modelli concettuali e strutture DB(Dev)/ collezioni documenti (HYM)
  • Architettura: Multi strati
  • Contenuto: molto buono (Dev)/ limitato (HYM)
  • Navigazione: limitato (Dev) / molto buono (HYM)
  • Presentazione: fixed format (Dev) / template language (HYM)

Un criterio di scelta

Applicazioni di dimensioni ridotte: EGS oppure IPM, eventualmente con una integrazione con una soluzione multi paradigma (MUP)

Applicazioni di medie dimensioni: DBS oppure GEN (INTRANET e B2B), altrimenti WDB o MUP

Applicazioni su larga scala: WBD oppure MUP, ma ci sono margini di miglioramento soprattutto per qunto riguarda GEN
