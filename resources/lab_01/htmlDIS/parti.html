Classi astratte

Sono a metà strada fra le interfacce e le classi:

interfacce
solo definizioni di metodi
classi
tutti i metodi devono essere implementati
classi astratte
si possono mettere sia metodo implementati che definizioni senza implementazione

Una definizione di metodo senza implementazione si chiama metodo astratto 

abstract class Astratta {
  int x;

  int esempio(int x) {
    return x*2;
  }

  abstract int altro(int x);
}

Servono per definire interfacce in cui alcuni metodi si possono già implementare

Implementare una classe astratta

Quasi tutto come per le sottoclassi

Però tutti i metodi astratti devono essere implementati 

class Prova extends Astratta {
  int altro(int x) {
    return x;
  }
}

Una classe può essere sottoclasse di una sola classe (astratta oppure no)

Una classe astratta può essere anche sottoclasse di una altra classe (astratta o no) e implementare diverse interfacce)

Metodi final

Sono metodi per i quali non si può fare overriding 

class Sopra {
  final int metodo() {
    return 0;
  }
}

Una sottoclasse di Sopra non può ridefinire metodo() 

class Sotto extends Sopra {
  // viene dato errore
  int metodo() {
    return 0;
  }
}
