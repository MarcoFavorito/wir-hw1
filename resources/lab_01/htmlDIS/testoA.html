Appello del 5/7/2004 - Compito A

Si vogliono gestire le prenotazioni dei biglietti per un determinato evento (musicale, sportivo, ecc.). Di un oggetto PrenotazioniEvento sono di interesse il nome dell'evento (una stringa) a cui si riferiscono le prenotazioni, ed una lista di prenotazioni. Di ciascuna  prenotazione è necessario sapere le seguenti informazioni:

  • nome della persona che effettua la prenotazione (una stringa). Si assuma che il nome sia identificativo della prenotazione, cioè che non possano esistere due prenotazioni aventi lo stesso nome nella lista;
  • numero dei biglietti prenotati (un intero).
Gli oggetti PrenotazioniEvento supportano le seguenti funzionalità:
  • crea: che, data una stringa n che rappresenta il nome di un evento, crea un oggetto PrenotazioniEvento relativo all'evento n per il quale inizialmente non esistono prenotazioni di biglietti;
  • nomeEvento: che restituisce il nome dell'evento a cui si riferiscono le prenotazioni;
  • inserisci: che dato il nome di chi effettua la prenotazione, ed il numero di biglietti richiesti, inserisce una nuova prenotazione in coda a quelle già presenti; tuttavia, se è  già presente una prenotazione con lo stesso nome, lancia una eccezione;
  • presente: che, dato il nome di una persona, restituisce true se fra le prenotazioni  è presente una prenotazione effettuata a quel nome, false altrimenti;
  • quantiBiglietti: che, dato un nome, restituisce il numero di biglietti indicato nella prenotazione effettuata a quel nome; se non esiste una prenotazione con il nome dato, genera una eccezione;
  • successore: che dato un nome n, restituisce il nome della persona che ha effettuato la prenotazione successiva alla prenotazione fatta da n; nel caso che la prenotazione effettuata da n sia l'ultima restituisce null; infine, se non è presente una prenotazione effettuata da n, lancia una eccezione;
  • elimina: che, dato un nome, elimina la prenotazione effettuata a quel nome; se la prenotazione non è presente, genera una eccezione;
  • listaPrenotazioni: che restituisce una struttura collegata formata da nodi della classe 
    public class nodoLista
{
    public String nome;
    public nodoLista next;
}
    in cui sono memorizzati i nomi di tutti coloro che hanno effettuato prenotazioni, nell'ordine in cui tali prenotazioni sono state effettuate.

Domanda 1. Scrivere una classe Java PrenotazioniEvento per rappresentare oggetti PrenotazioniEvento. Fornire il costo in tempo dei metodi che realizzano le funzionalità inserisci, elimina e listaPrenotazioni, motivando la risposta (indicando esplicitamente i parametri di input della funzione di costo ed il caso peggiore).

Domanda 2. Realizzare un metodo statico AssegnaBiglietti cliente della classe PrenotazioniEvento che, dati

  • un riferimento p ad un oggetto di classe PrenotazioniEvento,
  • un intero nb, che indica il numero di biglietti da assegnare,
  • il nome fout di un file

scriva su ciascuna riga del file fout il nome di chi ha effettuato la prenotazione, ed il numero di biglietti prenotato, per tutte le prenotazioni di p che possono essere soddisfatte avendo a disposizione nb biglietti, e nell'ultima riga del file scriva il numero dei biglietti rimanenti. Si noti che le prenotazioni vengono soddisfatte nell'ordine in cui esse sono state effettuate. Quindi, una prenotazione pr è soddisfatta se tutte le prenotazioni che precedono pr sono state a loro volta soddisfatte, e se il numero di biglietti richiesto da pr è inferiore o uguale al numero di biglietti ancora da assegnare. Fornire il costo in tempo del metodo realizzato, motivando la risposta (indicando esplicitamente i parametri di input della funzione di costo ed il caso peggiore).

Domanda 3. Spiegare la rappresentazione collegata di alberi binari in cui l'informazione in ciascun nodo è un riferimento ad un oggetto. Realizzare un metodo statico pubblico VerificaNonLineare, che, dato il riferimento alla radice di un albero binario alb,  restituisca true se l'albero è un albero binario in cui ogni nodo non foglia ha esattamente due figli, false altrimenti. Fornire il costo in tempo e spazio di memoria del metodo realizzato, motivando la risposta (indicando esplicitamente i parametri di input della funzione di costo ed il caso peggiore).
