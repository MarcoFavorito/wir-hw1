Problema 1 (18 punti)

I piatti offerti da un ristorante sono memorizzati in un file di testo (ad esempio menu.txt):

  • per ogni piatto, le seguenti informazioni sono memorizzate su una singola linea del file menú:

    1. nome (stringa di al massimo 30 caratteri);
    2. richieste (numero di volte che quel piatto è stato ordinato finora).

  • nel file menú i piatti sono riportati ordinatamente, dal meno richiesto al più richiesto.

Alla fine della giornata, tutti i piatti richiesti sono stati memorizzati in un file testo (ad esempio richieste.txt). Per ogni piatto richiesto durante la giornata, in questo file di richieste viene indicato il nome. Un nome per linea.

Nel seguito immagineremo che i piatti del menú vengano memorizzati in memoria centrale mediante una lista (rappresentata mediante record e puntatori).

  1. Definire i tipi di dato Pascal adeguati a risolvere i problemi di cui ai successivi punti 2) e 3). Chiamare TIPOLISTAMENU la lista dei piatti offerti dal ristorante.

  2. Siamo a fine giornata e dobbiamo aggiornare la lista menú, aggiungendo al numero di richieste di ciascun piatto, le richieste fatte oggi.

    Scrivere un sottoprogramma Pascal, ESER2, che, ricevendo

    • una lista menú LM
    • e il nome di un file di richieste nmf in cui si suppone siano contenuti i piatti richiesti oggi,

    leggendo dal file i piatti richiesti, aggiorni i campi richieste in LM.

  3. Adesso vogliamo salvare in un file menú i dati aggiornati con le richieste di oggi.

    Scrivere una procedura Pascal ESER3 che, ricevendo

    • il nome di un file di testo nmf (sarà il nuovo file menú),
    • una lista menú LM già aggiornata con i dati di oggi

      (che, quindi non è detto sia ordinata per richieste),

    scarichi in nmf i dati contenuti in LM, un piatto per linea (con relativo numero di richieste) facendo in modo che i piatti siano memorizzati in ordine di richieste (dal meno richiesto al più richiesto).

    suggerimento: se si vuole, si può costruire una copia ordinata di LM mediante inserimenti ordinati, scaricarla nel file e poi deallocarla.

Problema (a) (4 punti) Relativamente alla struttura di PILA, descrivere le operazioni fondamentali e la politica di gestione dei dati, mediante un esempio. Illustrare i diversi tipi di rappresentazione in PASCAL, mostrando per ciascuna rappresentazione l'implementazione di un'operazione base non banale.

Problema (b) (4 punti) Spiegare in cosa consiste un'istruzione macchina. Discutere i dispositivi costituenti una CPU e mostrare come la CPU esegue un programma scritto in linguaggio macchina.

Problema (c) (4 punti) Sia MATRIX una matrice di NxM numeri interi. Definiamo che MATRIX è a scacchiera se contiene zeri e numeri diversi da zero in modo alternato, come nella figura sotto: in particolare, le righe di indice dispari iniziano con uno zero; le righe di indice pari iniziano con un numero diverso da zero.

Scrivere una unità di programma FORTRAN (denominata ESERFORTRAN) che, ricevendo

-
una matrice MAT di NxM numeri interi,
-
il nome di un file NMF
-
e quanti altri parametri si ritenga opportuno,

  1. restituisca .TRUE. se MAT è a scacchiera ( .FALSE. altrimenti) e
  2. stampi nel file NMF la matrice, stampando però 1 al posto dei numeri diversi da zero.

Scrivere inoltre un esempio di main program contenente un'invocazione dell'unità progettata. In questo main program, comunque, le unità ulteriori che dovessero servire non vanno implementate: basta specificarne le intestazioni (nome e parametri).
