Aula

1)Cosa è Bison

E' un generatore di analizzatori sintattici; permette la generazione automatica delle specifiche C di un analizzatore sintattico. 

2)Come è possibile utilizzarlo

Perché Bison generi un analizzatore sintattico corretto, è necessario fornirgli le regole che governano la struttura sintattica del linguaggio.La prima cosa da fare è quindi descrivere il linguaggio mediante una grammatica non contestuale e stilare una specifica sintattica da sottoporre a Bison per l'interpretazione.In secondo luogo è necessario realizzare un analizzatore lessicale, che si preoccupi di leggere la sequenza di caratteri in ingresso e suddividerla in token; è possibile allo scopo utilizzare Flex, già visto negli incontri precedenti.

3)La specifica Bison

E' composta da tre sezioni distinte, separate dal simbolo %%

Dichiarazioni

%%

Regole della grammatica

%%

funzioni ausiliarie

Per rendere meglio l'idea, faremo riferimento in seguito alle specifiche Flex e Bison di esempio, stilate per la realizzazione dell'analizzatore del Pascom, fornito insieme al materiale.

/*

* Analizzatore lessicale per PASCOM (nome di questo file = pas.fl)

* (da usare insieme ad un analizzatore sintattico realizzato con BISON,

* ad esempio i file pas0.y, pas1.y, pas2.y)

*/

%{

#include <stdio.h>

#include "pas_tab.h"

%}

/* regular definitions */

delim[ \t\n]

ws{delim}+

letter[a-zA-Z]

digit[0-9]

id{letter}({letter}|{digit})*

number{digit}+

%%

{ws};

\{[^\}]*\};

(PROGRAM|program){return(Program); }

(BEGIN|begin){ return(Begin); }

(END|end){ return(End); }

(IF|if){ return(If); }

(THEN|then){ return(Then); }

(ELSE|else){ return(Else); }

(READ|read){ return(Read); }

(WRITE|write){ return(Write); }

(WHILE|while){ return(While); }

(DO|do){ return(Do); }

\;{ return(SEQOP); }

{number}{ return(NUM); }

":="{ return(ASSOP); }

[\+\-\*/]{ return(ARITOP); }

("="|">"|"<"|"<="|">="|"<>") { return(RELOP); }

{id}{ return(ID); }

%% 

/*

* Analizzatore sintattico per PASCOM

* (versione che stampa la sequenza delle regole grammaticali impiegate

* per effettuare la riduzione della stringa di input all'assioma

* della grammatica)

*

* Questo file (pas1.y) e'da usare insieme all'analizzatore sintattico 

* realizzato con FLEX a partire da pas.fl.

*

* Sequenza di comandi per generare l'analizzatore sintattico:

*

*flex pas.fl

*bison -d pas1.y

*gcc lexyy.c pas1_tab.c -lfl -o pascom

*

* Richiamare l'analizzatore sintattico tramite il comando:

*

*pascom < nomefileinput

*

* oppure ridirigendo l'output su un file:

*

*pascom < nomefileinput > nomefileoutput

*/

%{

#include <stdio.h>

#include <string.h> 

%}

%token Program If Then Else Begin End Read Write While Do SEQOP ASSOP 

ID RELOP ARITOP NUM

%start inizio

%%

inizio: Program ID SEQOP statement

{printf("inizio -> Program ID SEQOP statement\n");

}

;

statement:Begin statement rest_statements

{ printf("statement -> Begin statement rest_statements\n"); }

|ID ASSOP expression

{ printf("statement -> ID ASSOP expression\n"); }

|If test Then statement Else statement

{ printf("statement -> If test Then statement Else statement\n");

}

|While test Do statement

{ printf("statement -> While test Do statement\n"); }

|Read ID

{ printf("statement ->Read ID\n"); }

|Write expression

{ printf("statement -> Write expression\n"); }

;

rest_statements: SEQOP statement rest_statements

{ printf("rest_statements -> SEQOP statement rest_statements\n"); }

| End

{ printf("rest_statements -> End\n"); }

;

test: expression RELOP expression

{ printf("test -> expression RELOP expression\n"); }

;

expression: constant

{ printf("expression -> constant\n"); }

| constant ARITOP expression 

{ printf("expression -> constant ARITOP expression\n"); }

;

constant: NUM

{ printf("constant -> NUM\n"); }

| ID

{ printf("constant -> ID\n"); }

;

%% 

yyerror (s) /* Called by yyparse on error */

char *s;

{

printf ("SYNTAX ERROR\n", s);

}

main ()

{

yyparse ();

}

4)Le dichiarazioni

Innanzi tutto, nella sezione dedicata alle dichiarazioni possiamo introdurre dei commenti delimitati da "/*" e "*/" che verranno riportati interamente nel sorgente C generato.Possiamo utilizzare i commenti per spiegare dettagliatamente come, quando e perche' abbiamo realizzato la specifica in oggetto.

Analogamente ai commenti, verrà riportato sul codice generato, tutto ciò che nella sezione dichiarazioni viene racchiuso tra i simboli "%{" e "}%".Come mostrato nell'esempio, possiamo sfruttare questa possibilità ed utilizzare tali simboli per fare in modo che sul sorgente vengano riportate le direttive al preprocessore e la dichiarazione di eventuali variabili globali.

Nella sezione dedicata alle dichiarazioni vengono definiti anche i simboli terminali che verranno utilizzati.La dichiarazione avviene dopo la parola chiave %token; gli identificatori non dichiarati come token, verranno considerati simboli non terminali e pertanto dovranno comparire come parte sinistra di una regola.Dopo la parola chiave %start, viene indicato invece il simbolo iniziale della grammatica.In assenza di questa dichiarazione, viene considerato tale il simbolo corrispondente alla prima delle regole elencate.

5)Le regole

Nelle regole troviamo un insieme di coppie del tipo <regole della grammatica>-<azione semantica>.Le regole hanno la forma 

V: v1 | v2 | | vn;

Se un simbolo non terminale produce e si indica con la regola 

V: ;

Le cosiddette azioni semantiche vengono racchiuse tra i simboli { e }.Con esse è possibile specificare codice C da eseguire nel momento in cui viene applicata la regola associata.

6)   Le funzioni ausiliarie

Nella sezione dedicata alle funzioni ausiliarie, è possibile includere il codice C che implementa funzioni a cui si è fatto riferimento nella sezione delle regole.Il contenuto di tale sezione verrà riportato interamente nel modulo generato.Nell'esempio visto, viene implementata la funzione di gestione degli errori sintattici oltre al main() da riportare sul modulo C.

7)Come si genera l'analizzatore

La specifica sintattica generata deve essere elaborata da Bison.Il risultato di tale elaborazione è un modulo C denominato "<spec>_tab.c" al cui interno troviamo la funzione yyparse() che rappresenta l'implementazione effettiva dell'analizzatore.Questo va compilato contestualmente all'analizzatore lessicale generato con FlexL'eseguibile può essere richiamato specificando il file di input e il file di output.Se non specificata alcuna redirezione, vengono utilizzati lo standard input e output.Applicando quanto detto al nostro esempio e sapendo che la specifica lessicale e la specifica sintattica si trova nei file pas.fl e pas1.y possiamo generare l'analizzatore con i seguenti comandi:

flex pas.fl

bison -d pas1.y

gcc lexyy.c pas1_tab.c -lfl -o pascom

infine, per verificarne il funzionamento

pascom < nome_file_input > nome_file_output
