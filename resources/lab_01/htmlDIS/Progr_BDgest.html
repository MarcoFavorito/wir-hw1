Programma del corso

  1. Introduzione alle basi di dati
    • generalità; motivazioni; metodologie di progetto e standard industriali
    • dati e metadati, informazioni, conoscenza; stato della base di dati
    • linguaggi/modalità di accesso ai dati: DDL, DML, QL
    • modelli dei dati: caratterizzazione
  2. Sistemi di Gestione di Basi di Dati (DBMS)
    • generalità; motivazioni; utenti
    • funzionalità; architettura
  3. Metodologia di progetto di basi di dati
    • articolazione in fasi
    • documentazione di progetto
  4. Modello Entità-Relazione (ER)
    • strutture di base del modello ER
    • vincoli di chiave e di cardinalità; entità deboli
    • ruoli di relazioni
    • EER: gerarchie di generalizzazione
  5. Progetto concettuale
    • costruzione di schemi ER
    • criteri di completezza di schemi ER
    • esempi di progettazione concettuale
  6. Modello relazionale dei dati
    • aspetti strutturali; vincoli di identificazione e di integrità referenziale; viste; valori nulli
    • classificazione di linguaggi relazionali; cenni all'algebra relazionale e al calcolo sulle tuple
    • cenni alle forme normali (3 NF)
  7. Progetto logico
    • progetto logico ER-relazionale e ottimizzazione dello schema
    • cenni alla normalizzazione/decomposizione di schemi relazionali
  8. Linguaggio SQL
    • DDL (definizioni/modifiche), DML, definizioni di viste
    • interrogazioni semplici, nidificate, parametriche; trattamento della negazione
  9. Progetto fisico e ottimizzazione dei dati
    • motivazioni
    • tipologie di indice
    • indicizzazione primaria e secondaria
  10. Sviluppi delle tecnologie di trattamento dei dati
    • standard di rappresentazione dei dati
    • esempi di architetture applicative con DBMS
    • sviluppi ed evoluzione delle tecnologie per il trattamento dei dati
  • P_1. Un DBMS relazionale
  • P_2. Interfacciamento DBMS: ODBC, cenni JDBC
  • P_3. Esempi di architetture client-server (BD)
  • P_4. Progetto e sviluppo di applicazioni su DBMS relazionale

Risorse - tecnologie - ambienti di riferimento

È indispensabile la pratica di un DBMS relazionale. Fermo restando che può essere utilizzato un qualsiasi sistema, si segnalano, a titolo di esempio, Microsoft Access (in tutti i laboratori didattici della Facoltà), Microsoft SQL Server (laboratorio didattico, via Tiburtina), e il sistema FREEWARE mysql (http://www.mysql.com/).

Per fare pratica con il linguaggio SQL, si suggerisce l'utilizzo dell'ottimo programma (di cui esistono le versioni FREEWARE e trialware) WinSQL (HOME PAGE LINK o, più esattamente DOWNLOAD LINK). Questo programma è un client che deve essere utilizzato necessariamente con un DBMS relazionale.

Per avere un primissimo elementare esempio di costruzione di una base di dati realzionale, può essere di ausilio una piccola base di dati sviluppata in SQL a partire dallo schema di progetto.
