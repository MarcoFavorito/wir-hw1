Installing IOGRManager Tool

 

Getting IOGRManager Tool

The source code of IOGRManager Tool can be downloaded here (152Kb).

It consists of a package, named IOGRTool, containing:

  • IOGRUtil                       package containing all files exploited for creating and manipulating IOGRs.

  • Gui                              package containing theGUI.

  • ClientToolIOGR.java      Java file of a Client designed for the management of IOGRs.

  • ServerIOGR.java           Java file (servant) of the Server designed for the management of IOGRs.

  • IOGRManagerImpl.java Java file of IOGRManager implementation.

  • IOGR.idl                        IDL file containing the IOGRManager interface.

  • FT.idl                            IDL file containing the FT-CORBA extensions.

  • GIOP.idl                        IDL file of OMG.

  • IIOP.idl                         IDL file of OMG.

  • IOP.idl                          IDL file of OMG, with FT-CORBA extensions.

To install IOGRManager Tool, choose a directory and extract all files from the zip-archive there.

See the section Compiling IOGRManager on how to compile the source code and the section RunningIOGRManager on how to run the IOGRManager tool.

 

System Requirements

  • JDK 1.2 or later (http://java.sun.com).

  • CORBA ORB 2.4 compliant (JacORB1.3.30, Orbacus 4.0.5, Orbix2000).

  • Windows 98, NT, 2000.

 

 

Compiling IOGRManager

Before you can compile and run IOGRManager tool there is some additional set up you will need to do:

  • Assume IOGRManager tool is installed in C:\IOGRManager\. Add the following directories to your CLASSPATH:

    • C:\IOGRManager

    • C:\IOGRManager\IOGRTool

    • C:\IOGRManager\IOGRTool\Iogr

    • C:\IOGRManager\IOGRTool\org\omg\CORBA

    • C:\IOGRManager\IOGRTool\org\omg\CORBA\FT

    • C:\IOGRManager\IOGRTool\Gui

    • C:\IOGRManager\IOGRTool\IOGRUtil

  • If you want to use IOGRManager client, configure the Naming Service in the file properties of your specified ORB.

Now you can compile the source code in the following order:

  • Compile all IDL files (IOGR.idl, IOP.idl, GIOP.idl, IIOP.idl, FT.idl) with IDL to Java translator. The structure of IOGRTool package will be:

     

                      IOGRTool:

                                       -  Gui

                                       -  Iogr

                                       -  IOGRUtil

                                       -  org

                                       -  ClientToolIOGR.java

                                -  ServerIOGR.java

                                -  IOGRManagerImpl.java

                                -  IOGR.idl

                                -  IOP.idl

                                -  IIOP.idl

                                -  GIOP.idl

                                -  FT.idl

     

  • Compile all files generated by IDL to Java Translator (Iogr package, org.omg package and org.omg.CORBA.FT package).

  • Compile the package IOGRUtil and Gui.

  • Compile the package IOGRTool.

Now you can run the IOGRManager tool. For more informations on how to use the tool, see the section Running IOGRManager.

 

 

Running IOGRManager

After compiling evrything you're now ready to actually run the server and the client of IOGRManager. Make sure the Naming Service is running before starting either the server or the client.

You can now launch the server with the following command-line:

 

   java [ORB-properties] IOGRTool.ServerIOGR

 

The client can be started with the following command-line:

 

   java [ORB-properties] IOGRTool.ClientToolIOGR

Now you can create and modify IOGRs, using the operations of IOGRManager interface.

 

For example, if you want to create an IOGR for Hello example, you can first run two or more Hello server and then you can create an IOGR containing their object references.

Suppose that Server1 and Server2 are two Hello server registered into NamingService.

To create an IOGR containing these two references, you can open the menu File and select New IOGR:

 

 

 

 

This produces the opening of another window:

 

 

 

 

In this window you can choose:

  • The name of the object group (e.g. IOGRHello).

  • The value of TagFTGroup fields (e.g. MyDomain,1,1).

  • The names of the objects whose IIOP profiles has to be inserted into the IOGR you want to create.

In this case, we create an IOGR, named IOGRHello, with two Hello server, Server1 and Server2:

 

 

 

 

Clicking on the buttom Create IOGR you create the new IOGR, that appears in the IOGRManager window:

 

 

 

 

Now you can modify this IOGR:

  • Add new objects.

  • Remove one or more objects.

  • Set the primary member of the object group.

  • Set the version of the object group.

  • Delete IOGR.

  • ...etc.

and you can create other IOGRs.
