14) Il livello applicativo in Internet - CGI e sistemi Java-based

 1.  L'interfaccia CGI

    Specifica il protocollo di comunicazione, ovvero le regole che governano l'interfacciamento tra server web, browser e applicazioni CGI che affiancano il server http.

    In una pagina html è possibile introdurre URL che individuano risorse particolari come programmi residenti su server (solitamente nella cartella cgi-bin)

    <A HREF="http://nome_dominio/cartella_del_server/eseguibile.exe>…>

2.  Ciclo di esecuzione di una richiesta CGI
    • Il server http fornisce al browser una pagina html contenente (sotto forma di link) il riferimento ad un'applicazione CGI.
    • Il browser esegue una richiesta al server http, di invocazione (con eventuali parametri) dell'applicazione CGI.
    • Il server http esegue l'attivazione dell'applicazione CGI, redirezionando l'input standard e l'output standard.
    • L'applicazione CGI elabora la richiesta e fornisce la risposta (es: una pagina html).
    • Il server http fornisce il risultato prodotto al browser richiedente.
3.  Interazione con l'utente nelle pagine html

    L'html fornisce dei tag con i quali è possibile definire formati di input da proporre al visitatore (esempio: test1form.html). L'utilizzo di tali tag e la specifica CGI consentono la realizzazione di "ambienti" client/server su internet.

4.  Passaggio di parametri

    Le modalità per il passaggio di eventuali parametri dipendono dal metodo di richiesta utilizzato: "GET" o "POST". Nel metodo GET i parametri vengono accodati all'URL che individua il CGI, mentre nel metodo POST i dati vengono inseriti nel corpo della richiesta http. Il primo metodo soffre dei limiti dettati dalla lunghezza massima dell'URL (256 caratteri), con il metodo POST invece tale limite può essere abbondantemente superato. Nel metodo GET, un punto interrogativo sull'URL, separa l'identificativo della risorsa dai parametri, eventualmente (se più di uno) separati dal carattere "+" (test2get.html, test2getnopar.java, test2gethref.java, test2getgif.java). Nel metodo POST i dati vengono accodati all'header http e rigirati allo standard input dell'applicazione CGI (test3post.html, test3postpar.java).
